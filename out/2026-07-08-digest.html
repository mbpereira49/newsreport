sqlite-utils gets its first major bump since 2020

Version 4.0 adds schema migrations, nested transactions, and compound foreign keys—small plumbing with outsized practical value.

Simon Willison released sqlite-utils 4.0, calling it the project’s 124th release and its first major version bump since 3.0 in November 2020. The release includes “small but significant” breaking changes, plus three headline additions: database schema migrations, nested transactions through a new db.atomic() method, and support for compound foreign keys. The migrations feature is the main turn: sqlite-utils can now define a sequence of database changes in Python files, track which have been applied, and apply pending ones.

Why it matters: SQLite is often chosen because it avoids operational ceremony; first-class migrations make it easier to keep that simplicity while treating schema changes as real application infrastructure.

Sources: Simon Willison
